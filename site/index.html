API-first telemetry core for security & ops signals.

orbit-core ingests metrics and events from multiple sources (Nagios, Wazuh, n8n, Fortigate via Wazuh), stores them in Postgres with rollups + retention, and exposes a stable HTTP API for timeseries, events and multi-series group-by queries.

Why orbit-core

Most monitoring stacks drift into hard-to-maintain pipelines: metrics in one place, events in another, dashboards glued to brittle queries, and unbounded dimensions that explode cardinality. orbit-core is built to stay fast and predictable.

Unify metrics + events

Store both telemetry types in one canonical core and query them through a stable API.

Rollups + retention

Plain Postgres with scheduled rollups. Target: 180 days retained (configurable).

Performance-first queries

Auto-bucket, safe limits and a source-table selection strategy (raw vs rollups).

Cardinality control

Top‑N limiting for group-by dimensions to prevent runaway series explosions.

How it works

Connectors ship deterministic batches to ingest endpoints. The API validates, stores and serves queries. Deploy behind TLS + auth and it works behind subpaths.

Core endpoints

  • GET /api/v1/health — build + health
  • GET /api/v1/metrics — JSON metrics
  • GET /api/v1/metrics/prom — Prometheus exporter
  • POST /api/v1/ingest/metrics — MetricPoints
  • POST /api/v1/ingest/events — Events
  • POST /api/v1/query — orbitql query
  • GET /api/v1/catalog/* — assets/metrics/dimensions
  • POST /api/v1/dashboards/validate — validate DashboardSpec (builders)

Storage strategy

metric_points raw ≤ 14d
metric_rollup_5m downsample ≤ 90d
metric_rollup_1h downsample ≤ 180d
Query engine selects the source table automatically based on the requested time range.
Want details? See docs/ARCHITECTURE.md.

Quickstart

Local dev in a few minutes (excluding Postgres image pull).

Terminal
git clone https://github.com/rmfaria/orbit-core
cd orbit-core

pnpm install

# Postgres (optional)
docker compose -f scripts/dev-postgres.docker-compose.yml up -d
export DATABASE_URL='postgres://postgres:postgres@localhost:5432/orbit'

pnpm dev

# API: http://localhost:3000/api/v1/health
# UI:  http://localhost:5173

Production note

Keep secrets server-side. Put orbit-core behind a reverse proxy with TLS and auth. You can also enable an application API key (see ORBIT_API_KEY). Subpath deployments are supported.

Connectors

Connectors are deterministic, cron-friendly and no-AI.

Primary

Nagios Connector (Python)

Ships perfdata and HARD state changes (metrics + events).

Install: connectors/nagios/INSTALL.md

Ready

Wazuh Connector (Python)

Ships Wazuh alerts from alerts.json into orbit-core events (severity mapping + state cursor).

Install: connectors/wazuh/INSTALL.md

Ready

n8n Connector (Python + workflow)

Ships n8n execution errors + stuck runs (polling API) and optional realtime Error Trigger workflow.

Install: connectors/n8n/INSTALL.md

Via Wazuh

Fortigate (syslog → Wazuh)

Fortigate logs flow through Wazuh and are surfaced as namespace=wazuh + kind=fortigate.

Install: connectors/fortigate/INSTALL.md

Optional

Nagios shipper (TypeScript)

Monorepo-integrated shipper package (use if you prefer TS toolchains).

Docs: packages/nagios-shipper

Security

  • No secrets in the browser (server-side only).
  • Prefer TLS + auth in front of the API.
  • Use safe limits and avoid unbounded queries.
Security policy: SECURITY.md

Roadmap
